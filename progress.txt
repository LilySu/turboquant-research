## RaBitQ Key Concepts Summary

### Core Algorithm
1. **Normalize** data vectors to unit hypersphere (centered on centroid)
2. **Codebook** is 2^D bi-valued vectors: C = {±1/√D}^D
3. **Random rotation** P converts C to C_rand = {Px | x ∈ C}
4. **Quantize** by finding nearest: signs of P⁻¹o give D-bit code
5. **Estimate** inner product: ⟨ō,q⟩/⟨ō,o⟩ is UNBIASED
6. **Error bound**: O(1/√D) with high probability (asymptotically optimal)

### Key Differences from PQ
- PQ uses ⟨ō,q⟩ ≈ ⟨o,q⟩ which is BIASED
- RaBitQ divides by ⟨ō,o⟩ to correct bias
- RaBitQ has theoretical error bounds; PQ does not
- RaBitQ uses bitwise operations; PQ uses LUT lookups

### Efficient Computation
- **Bitwise**: AND + popcount for single vectors
- **FastScan**: SIMD with 4-bit LUT for batch of 32 vectors
